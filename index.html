1) Install / Include

2) Theme Variables (Optional but recommended)

Your components read a few CSS custom properties. Override them once to match your brand.

:root{
  --trans-black: rgba(0,0,0,.18);
  --text-color: #0f172a;        /* text on components */
  --element-bg: #ffffff;        /* surface color for .def, .card */
  --border-col: #d0d5dd;        /* borders for .bor */
  --none: transparent;          /* used by .trans and .ico-btn */
}

3) Spacing Utilities

Padding utilities add space inside an element. Numbers are in pixels. Example: p-l-16 = padding-left:16px.

‚Ä¢ Left Padding ‚Äî p-l-[1..100]

<div class="p-l-40 demo-box">This box has 40px left padding</div>

‚Ä¢ Right Padding ‚Äî p-r-[1..100]

<div class="p-r-20 demo-box">This text moves away from the right edge by 20px</div>

‚Ä¢ Bottom Padding ‚Äî p-b-[1..100]

<button class="btn p-b-12 bor">Extra bottom padding (12px)</button>

Beginner tip: If you need top padding, wrap with a container and use p-b-* on a child, or add a top utility in your CSS similar to the others (pattern is the same).

9) Common Mistakes & Tips

  • Spacing doesn‚Äôt work? Check that you used the right side: p-l- for left, p-r- for right, p-b- for bottom.
  • Gap not working? Make sure the parent has a flex class like .r or .c.
  • Colors look off? Override the CSS variables in :root to match your theme.
  • Need top padding? Add a p-t-* utility following your pattern, or wrap children and use p-b-* strategically.
  • Icon-only buttons: use .ico-btn + .bor for a minimal affordance and set aria-label.
